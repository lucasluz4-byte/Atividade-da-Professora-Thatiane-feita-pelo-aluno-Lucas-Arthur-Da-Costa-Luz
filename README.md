**Criação do repositórioerve para:**

Criar um local online onde o código do projeto ficará armazenado.

Permitir controle de versões do código.

Facilitar colaboração entre várias pessoas.

Exemplo: criar um repositório chamado projeto-site.

**Clone do repositório**

Serve para:

Baixar uma cópia do repositório do GitHub para o computador.

Permitir que o desenvolvedor trabalhe no projeto localmente.

Facilitar a edição dos arquivos do projeto.


**Criação de branch**

Serve para:

Criar uma linha separada de desenvolvimento.

Permitir trabalhar em novas funcionalidades sem alterar o código principal.

Organizar melhor o desenvolvimento do projeto.


**Registro de commits**

Serve para:

Registrar as alterações feitas no projeto.

Guardar o histórico de modificações do código.

Permitir voltar para versões anteriores caso seja necessário.


**Envio das alterações (push)**

Serve para:

Enviar as alterações do computador para o repositório no GitHub.

Atualizar o projeto online.

Compartilhar as mudanças com os outros membros da equipe.


**Atualização do repositório local (pull)**

Serve para:

Baixar as atualizações feitas no GitHub.

Manter o repositório local atualizado.

Integrar alterações feitas por outros colaboradores.


**Criação de Issues**

Serve para:

Registrar problemas encontrados no projeto.

Criar tarefas que precisam ser realizadas.

Organizar as atividades da equipe.


**Organização com Milestone**

Serve para:

Agrupar várias issues relacionadas.

Definir metas ou etapas importantes do projeto.

Acompanhar o progresso até uma determinada versão.


**Gerenciamento com Project**

Serve para:

Organizar as tarefas do projeto em um quadro.

Controlar o andamento das atividades.

Visualizar o que está pendente, em andamento ou concluído.
