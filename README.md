*Criação do repositórioerve para:*

Criar um local online onde o código do projeto ficará armazenado.

Permitir controle de versões do código.

Facilitar colaboração entre várias pessoas.

Exemplo: criar um repositório chamado projeto-site.

**Clone do repositório**

Serve para:

Baixar uma cópia do repositório do GitHub para o computador.

Permitir que o desenvolvedor trabalhe no projeto localmente.

Facilitar a edição dos arquivos do projeto.


**Criação de branch**

Serve para:

Criar uma linha separada de desenvolvimento.

Permitir trabalhar em novas funcionalidades sem alterar o código principal.

Organizar melhor o desenvolvimento do projeto.


**Registro de commits**

Serve para:

Registrar as alterações feitas no projeto.

Guardar o histórico de modificações do código.

Permitir voltar para versões anteriores caso seja necessário.
